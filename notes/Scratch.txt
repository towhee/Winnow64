⏺ Done. After rebuildMaskTools() finishes (which runs on every expand/collapse/add/delete via setSelectedMask), it now calls
  viewport()->update() to force a full content repaint.

  The cause: the header-style tool rows draw their caption across the full row width (into the value column via the delegate's r4 = w0 +
  w1). On collapse the view only repaints a narrow dirty region, so the caption gets clipped to column 0 — exactly what you saw — until a
  hover triggers a full repaint. Forcing a whole-viewport update reproduces that full repaint immediately, so the captions stay full-width.

  This is a one-line, low-cost fix (one coalesced repaint per rebuild). Please rebuild and confirm the expand→collapse no longer truncates.
