To let the TSan app read the USB (/Volumes/Untitled), grant Full Disk Access to the Terminal app you launch the script from:

  1. Open System Settings (Apple menu  → System Settings).
  2. Go to Privacy & Security → Full Disk Access.
  3. Click the + button (you may need to authenticate with Touch ID / password).
  4. In the file picker, navigate to Applications → Utilities → Terminal.app (or iTerm.app if you use that) and add it. Alternatively, drag
  Terminal in from Finder.
  5. Make sure its toggle is on (blue).
  6. Quit Terminal completely (Cmd-Q) and reopen it — the permission only applies to newly launched processes.

  Then verify and run:

  ls /Volumes/Untitled            # should now list DCIM etc., not "Operation not permitted"
  cd /Users/roryhill/Projects/Winnow64
  WINNOW_TSAN_FOLDER='/Volumes/Untitled' tests/tsan/run_tsan_proxy.sh

  Notes:
  - If you'd rather not grant blanket Full Disk Access, Privacy & Security → Files and Folders sometimes lists Terminal with a Removable
  Volumes toggle — enabling that is narrower. But Full Disk Access is the reliable one for /Volumes.
  - The Winnow GUI app you normally run already has whatever access it was granted on first launch; this step is only needed because the
  TSan build runs as a child of Terminal, which inherits Terminal's permissions.
  - If ls still fails after re-opening Terminal, double-check you added the right Terminal (the one actually running the command — echo
  $TERM_PROGRAM will say Apple_Terminal or iTerm.app).

  Once ls /Volumes/Untitled works, the script will load the real ORF tree, and Pass C will tell us whether the QSortFilterProxyModel mapping
  race is real.
