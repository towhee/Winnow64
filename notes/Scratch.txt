progressbar.cpp was initially designed to show the status of ImageCache.  It has a fixed
height.  Over time I have added the MetaRead and FocusStack progress.  I would like to
replace ProgressBar with a new class called Progress:

- One container that can hold many progress rows. Each row has two columns: text and
progress bar. The text font size should have a separate variable that can be changed. The
container width should be adjustable, and the contained progress bars fit the container
width. The height is variable to fit the rows. The container background is the same as
MW::statusBar.

- each row has its own update function that defines the text, pixel height of the bar,
color of the bar and custom display (for example ProgressBar::updateImageCacheProgress).
Each row height height fits the text and bar heights.

- container function to hide/show a bar.  Adjust MW::statusBar height to fit Progress.

- new rows are added by adding a new update function adding to any container objects that
have been created to manage Progress.

- if feasible, design Progress so it can be located in MW::statusBar or in a separate
window if it gets too large.

- Retain ProgressBar until later in case this does not work out.
